# StockDetailV2 Blueprint v1.0

## 目标

StockDetailV2 是个股交易复盘页。它不替代旧版匹配算法，只复用现有算法结果，重新组织展示层。

核心目标：

1. 看清这只股票当前还有什么未匹配。
2. 看清历史匹配收益来自哪些买卖闭环。
3. 支持买后卖、先卖后买。
4. 手机端优先。
5. 默认折叠，避免流水账。

## 设计原则

1. 不改旧版匹配算法。
2. 不删除旧版详情页。
3. V1 和 V2 并存。
4. V2 只做展示适配。
5. 手动数据可能只有日期，没有成交时间。
6. 所有展示按日期降级。
7. 默认折叠，点击后展开。
8. 不显示无意义的 0 值模块。
9. 盈利用红色，亏损用绿色。
10. 先 Blueprint，后 Coding。

## A 股颜色规范

- 盈利 / 正收益：红色
- 亏损 / 负收益：绿色
- 未匹配买入：蓝色
- 未匹配卖出 / 待买回：紫色
- 部分匹配：橙色
- 中性文字：灰色

## 数据适配函数

`buildStockDetailData(stockCode)` 输出：

```js
{
  stockCode,
  stockName,
  currentPrice,
  currentHolding,
  unmatchedDirection: "buy" | "sell" | "none",
  unmatchedQty,
  totalMatchedQty,
  totalMatchProfit,
  dateGroups: [],
  unmatchedLots: [],
  rawTrades: []
}
```

## 未匹配方向规则

- 剩余买入数量大于 0：`unmatchedDirection = "buy"`，页面显示“未匹配买入”。
- 剩余卖出数量大于 0：`unmatchedDirection = "sell"`，页面显示“待买回”。
- 没有剩余：`unmatchedDirection = "none"`，页面显示“无”。
- 禁止显示负数未匹配数量。
- 禁止同时显示“未匹配买入 0”和“待买回 18,900”。

## 顶部统计卡片

显示 4 张卡片：

1. 当前持仓
2. 未匹配方向
3. 未匹配数量
4. 累计匹配收益

## Tab 结构

1. 匹配
2. Lot
3. 图表
4. 交易
5. 概览

默认进入“匹配”。

## 匹配 Tab

- 顶部显示匹配进度。
- 日期分组按日期倒序。
- 默认只显示最近 10 个日期组。
- 最近日期组默认展开，其他日期组默认折叠。
- 触发卡片默认折叠。
- 展开后只显示前 2 条匹配，支持“查看全部”。
- 不渲染空模块。

## Lot Tab

Lot Tab 只显示当前未匹配方向。

- `unmatchedDirection = "buy"`：显示未匹配买入 Lot。
- `unmatchedDirection = "sell"`：显示待买回卖出 Lot。
- `unmatchedDirection = "none"`：显示暂无未匹配 Lot。
- 默认显示最近 10 条，支持加载更多。

## 图表 Tab

先保留占位：

1. 仓位变化
2. 成本变化
3. 匹配收益

不为了图表改算法。

## 交易 Tab

显示该股票原始成交记录，按日期倒序。每条显示日期、买/卖、数量、价格、金额。

## 概览 Tab

显示一句话总结：当前持仓、当前未匹配方向、累计匹配收益、最近匹配集中日期。

## 移动端排版

1. 顶部统计卡片 2 列。
2. 日期组纵向单列。
3. 日期组默认折叠。
4. 触发卡片默认折叠。
5. 卡片间距足够。
6. 一屏至少能看到多个日期摘要。
7. 不要出现几百行流水账。

## 开发范围

允许修改：

- StockDetailV2 展示函数
- StockDetailV2 CSS
- buildStockDetailData
- V2 Tab 渲染逻辑
- V2 日期分组渲染逻辑

禁止修改：

- 旧版匹配算法
- 旧版详情页
- 导入逻辑
- 成交记录原始数据结构
- Dashboard V1
- Calendar V1

3. 顶部显示统一未匹配方向。
4. 日期组默认折叠。
5. 最近日期默认展开。
6. 默认只显示最近 10 个日期组。
7. 可以加载更多。
8. 触发卡片默认折叠。
9. 展开后只显示前 2 条匹配，支持查看全部。
10. 空模块不显示。
11. 盈利用红色，亏损用绿色。
12. Lot Tab 只显示当前未匹配方向。
13. 一屏不能变成流水账。
