# A股交易分析 iPhone 工具要求

本文档记录当前 iPhone 版 A 股交易分析工具的核心要求。以后继续修改时，应优先遵守这些原则，避免破坏已经满意的功能和使用体验。

## 基本原则

- 工具优先服务 iPhone 使用场景。
- 页面必须适合手机单手操作，减少长距离下拉和横向拖动。
- 所有功能先在本地文件中试用，用户满意后再上传 GitHub。
- 未经用户明确说“上传”，不要推送到 GitHub。
- 原有稳定功能不要随意重构或改变计算逻辑。
- 修改排版时只改视觉和布局，不改变数据计算。
- 中文界面为主，表达要直观、清楚。

## iPhone 界面要求

- 页面宽度必须完美匹配 iPhone 屏幕，不留左右空白。
- 在电脑侧边栏预览时，也要贴合当前预览宽度，不出现右侧空白。
- 底部 Tab 固定在底部，按钮要足够大，适合手指点击。
- Tab 名称前可以使用图案，让入口更直观。
- 总览、收盘、交易、持仓、流水、总结、复盘、设置等入口都应清楚可见。
- 长内容应优先使用弹窗、折叠、日期查询或最近记录方式展示。
- 不要让用户因为记录越来越多而一直下拉很久。
- 重要卡片之间要紧凑但不拥挤。
- 股票名称需要醒目，目前倾向红色粗体配黄色底。
- 中国习惯：红色代表盈利，绿色代表亏损。

## 弹窗要求

- 所有弹窗必须匹配 iPhone 全屏宽度。
- 弹窗不要留左右侧边，也不要在顶部留空。
- 弹窗里的“标题 + 关闭”行要固定在顶部。
- 用户滚动弹窗内容时，关闭按钮必须一直可见。
- 弹窗层级要高于底部 Tab，避免被底部导航挡住。
- 适合放入弹窗的内容包括：
  - 各股仓位
  - 任意区间盈亏
  - 各股匹配盈亏
  - 分红明细
  - 个股作战计划
  - T 表现
  - 持股不动 vs 低买高卖
  - 未匹配明细
  - 匹配明细
  - 完整交易时间线
  - 初始持仓明细

## 数据与存储要求

- 数据必须保存在本机浏览器本地存储中。
- iPhone 添加到主屏幕后，应支持离线打开。
- 必须保留 `manifest.webmanifest`。
- 必须保留 `service-worker.js`。
- 离线状态下可以查看和录入本地数据。
- 股价更新需要网络。
- 不要依赖电脑运行。
- iPhone 添加到主屏幕后，应能作为独立固定版使用。
- 用户应定期导出 JSON 备份，但总览页不显示备份提醒卡。

## GitHub 要求

- 当前新版仓库为 `lin-1234y/ashare-trading999`。
- 只有用户明确说“上传”时，才推送到 GitHub。
- 上传前必须确认：
  - 只提交当前需要的文件。
  - JavaScript 语法检查通过。
  - `manifest.webmanifest` 和 `service-worker.js` 仍存在。
  - 离线缓存名不要与旧仓库冲突。
- 如果用户说不满意，应先在本地改，不要急着上传。

## 核心功能要求

- 手动输入买卖交易数据。
- 支持设置佣金、印花税、过户费。
- 支持输入初始现金。
- 支持输入、编辑、删除初始持仓。
- 初始持仓应有明细弹窗。
- 支持清空全部初始持仓。
- 支持股票名称自动填充。
- 支持手动更新股价按钮。
- 股价自动更新不应干扰用户对当日盈亏的手动记录。
- 支持记录每日收盘。
- 每日收盘记录应支持日期或日期区间查询。
- 每日收盘应记录总资产，也要能看到各股明细。
- 资产曲线应保留日历，并生成折线图。
- 资产曲线可计算最大回撤。

## 盈亏计算要求

- 总盈亏、当日盈亏、月度盈亏、年度盈亏必须以账户资产变化为核心。
- 总收益率应基于总资产和原始资产。
- 当月收益率应为：当月收益 / 月初资产。
- 匹配实现盈亏必须与资产盈亏分开。
- 匹配实现盈亏用于衡量低买高卖，不代表全部账户盈亏。
- 原始持仓不参与低买高卖匹配。
- 匹配支持先卖后买。
- 匹配按最近原则进行。
- 已匹配数量、未匹配数量、匹配实现盈亏要清楚显示。
- 持股不动 = 总盈亏 - 匹配实现盈亏。
- 个股详情中应展示：
  - 原始成本
  - 现在成本
  - 当前数量
  - 当前市值
  - 手续费
  - 低买高卖
  - 总盈亏
  - 总收益率
  - 已匹配数量
  - 未匹配数量
  - 当日盈亏
- 匹配数量标签不要使用红色背景，字体可用绿色。

## 分红与送股要求

- 支持现金分红。
- 支持送股 / 转增。
- 设置栏里公司行为应可添加、编辑、删除。
- 现金分红填写每股红利或金额时，应能算出净分红。
- 分红税由用户自行填写。
- 净分红是进入账户盈亏计算的锚。
- 总览中显示累计净分红。
- 总览中应有分红明细弹窗。
- 分红明细弹窗显示个股分红日期和税前分红记录。

## 复盘与总结要求

- 总结页面应先整体，后个股。
- 支持周、月、季、年总结。
- 总结要看账户资产变化，而不是孤立交易数字。
- 个股表现应尽量与持仓页逻辑一致。
- 复盘报表里个股日历应显示：
  - 当日盈亏
  - 匹配实现盈亏
  - 成交额
- 复盘报表日历金额取整数。
- 总览日历金额也取整数。
- 日历月份应可切换上月、本月、下月。
- 以后如果需要查看历史月份、年份，应保留年月选择能力。

## 视觉风格

- 当前方向为黑色背景。
- 内容可用金色、红色、黄色、蓝色等多色渲染。
- 总资产卡片使用偏蓝背景，金额使用金龙感的闪亮金色。
- App 图标使用真金龙风格。
- 整体感觉应美观、大气、适合 iPhone。

## 验证要求

- 每次修改后至少运行 JavaScript 语法检查。
- 重要排版修改后，尽量用手机尺寸截图检查。
- 检查重点：
  - iPhone 宽度是否无横向滚动。
  - 右侧是否无空白。
  - 底部 Tab 是否固定且不挡内容。
  - 弹窗是否全屏贴合。
  - 弹窗关闭按钮是否固定。
  - 文字是否重叠或被遮挡。
  - 功能是否仍能正常打开。
